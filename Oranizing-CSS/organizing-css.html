Working with CSS

Tips for Organizing CSS

  • Use comments liberally
  • User consistent naming conventions
    • As in how you name your selectors - are you using dashes, underscores, camel case?)
  • Use consistent formatting
    • Be consistent in your spacing, tabs, whether the rule is all on one line or on multiple lines, etc.
  • Group like styles whenever possible. For example, group all the h1, h2, etc. in the same area.
  • Split up files

Browser Inspection Tools

  • Right click on web page
  • Click Inspect
  • The CSS on this is a live editor, meaning you can try out CSS properties w/o saving them to the project.

Using the "Can I use..." for a CSS Property

  • W3.org CSS current work & how to participate
    • This is the official documentation for various specification modules, as well as their status
    • It lets you know if you can use a particular CSS property in a specific browser
    • This is a pretty dense site, so caniuse.com might be a better choice.
  • caniuse.com
    • Provides up-to-date browser support tables showing which frontend web technologies, like JavaScript, HTML5, and CSS, are available for use.
    • You can simply type in the feature you want to use, for example flexbox, and you'll get a table showing support across various browsers and browser versions.
    • There's also a statistic you can look at quickly to gauge how well a feature is supported.

Validating Your CSS with Tools

  • It's important that before you add your styles to a live website, you're sure everything is error-free and working as intended.
  • w3c.org CSS Validation Service
  • CSS LINT
