Structuring a Page with CSS

  • A browser reads an HTML document from the top down.
  • If there's no CSS intervention, that HTML will appear on the page as a bunch of stacked elements.

HTML Page Layouts

  • Tables
    • Before CSS, people just used tables to create row and column-based layouts. This was terrible as that's not really what HTML table markup was meant for and it left us with a mess of unsemantic HTML, inaccessible layours, and code taht was very difficult to maintain.
  • CSS Floats, positioning, and clears
    • Not really a layout system, but it did give us the ability to manipulate where elements appeared on a page relative to their neighboring elements.
  • CSS frameworks and grids
  • Carrie's calling all of these HTML layouts because they primarily rely on the actual structure of the HTML document to put content where you'd want it on the page.
  • Flexbox
    • Commonly Used Today
    • A one-dimensional flexible box module that lets us distribute content evenly and flexibly to fill all the available space.
  • CSS Grid
    • First true two-dimensional layout tool in CSS that lets us treat the viewport like a canvas.
    • Radically changes how we design page layouts for the web.

Flexbox (Flexible Box Layout Module)

  • A layout module for CSS that gives you an efficient way to arrange, organize, and size website elements to create responsive designs.
  • Lets us arrange items within a parent container and then get specific about how we want those items to behave in relationship to one another.

Flexbox

*CSS-Tricks: A complete Guide to Flexbox
  • Flex container (parent)
    • < section class="container >
      < /section >
  • Flex items (children) (elements within the parent container)
    • < section class="container >
      < /div >
      < /div >
      < /div >
      < /section >
    • The flex items are placed along flex lines, which are either a vertical or horizontal axis.
    • These items can be manipulated in layout, size, spaceing, and even change order along the flex line.
      *** You can manipulate the order of flex children without touching the HTML structure.
    • This allows you to take advantage of the available view port space and lets elements
      arrange themselves automatically as the view port dictates.
    • Alignment
    • Direction
    • Order
    • Size
  • Flexbox is a one dimensional presentation model. This means that layouts are processed in one dimension at a time, either as a row or a column.
  • You can define how space is distributed between elements and how they should be aligned in relationship to the parent container.

Flexbox Benefits

  • Responsive (mobile-friendly)
  • Ease of positioning child elements
  • Change presentation order without altering HTML
  • Control width and height based on parent element

Flexbox generally works best on small scale layouts.

CSS Grid

  • .container { display: grid; }
  • The most powerful way to control page layouts
  • A two-dimensional layout system. This allows developers to place HTML elements in a Grid-based layout using both rows and columns simultaneously.
  • With Grid, we can apply a grid to a container and then put content wherever we want within that grid - Completely independent of semantic HTML markup.
  • Equivalent of a print-based grid layout with the flexibility of responsive design.

CSS Grid Concepts and Terminology

  • .container { display: grid; }
  • A grid is an element with its display properties set to grid.
  • Grids have rows, and those are horizontal lines, and columns, which are vertical lines.
  • Grid Lines are the lines that make up the grid and they can be horizontal or vertical.
  • A Grid Track is the space between two grid lines. Grid tracks can also be horizontal or vertical, representing rows or columns.
  • A Grid Cell is the space between four Grid Lines.
  • A Grid Area is any area within the grid defined by its four outer most grid lines. A Grid Area can cover multiple lines and contain multiple cells.
  • You define a grid, and you either allow items to be automatically placed into cells defined by that Grid, or control their positioning.

CSS Framework (or Library)

Collection of style sheets that provide layout and styling for common design elements

A CSS framework is a collection of CSS files with predefined rules for commonly used things like typography, UI components, layouts, and browser resets.

  • Typography
  • Buttons, forms, controls
  • Grids/layouts
  • Resets

Benefits of Using a CSS Framework

  • Achieving cross-browser compatibility
  • Consistent styling
  • Creating responsive layouts
  • Speeding up your development time
  • Learning CSS by studying other people's code

Popular CSS Frameworks

  • Bootstrap
  • Tailwind

Responsive Web Design

An approach to web design used to create layouts that respond to the device used to view the content

CSS Gives Us Multiple Ways to Achieve This:

  • Media Queries
    • Can be used to check many things:
    • Width and height of a viewport
    • Width and height of the device
    • Orientation (portrait/landscape
    • Resolution
  • Flexbox and Grid
    • Responsive by default
    • In Flexbox, flex items will shrink and distribute space between the items according to the space available in their parent container. You can specify values to indicate how you want those items to behave when there's either more space or less space around them.
    • CSS Grid allows the distribution of available space in a container across grid tracks. So, grid responds natively to whatever the container size is.
  • Typography Responses
    • Using media queries, we can also make typography responsive
    • For example, if we have a base font size of 20px, we can then scale that up or down using rem units.
      • html { font-size; 20px; }
        h1 { font-size 2rem; }
        @media (min-width: 1200px) { h1 { font-size: 4rem;} }
    • Alternatively, you can use the view port width to dynamially calculate font size
      • <meta name="viewport"
        content="width=devide-width, initial-scale=1">
