Layouts

Structuring a Page with CSS

  • A browser reads an HTML document from the top down.
  • If there's no CSS intervention, that HTML will appear on the page as a bunch of stacked elements.

HTML Page Layouts

  • Tables
    • Before CSS, people just used tables to create row and column-based layouts. This was terrible as that's not really what HTML table markup was meant for and it left us with a mess of unsemantic HTML, inaccessible layours, and code taht was very difficult to maintain.
  • CSS Floats, positioning, and clears
    • Not really a layout system, but it did give us the ability to manipulate where elements appeared on a page relative to their neighboring elements.
  • CSS frameworks and grids
  • Carrie's calling all of these HTML layouts because they primarily rely on the actual structure of the HTML document to put content where you'd want it on the page.
  • Flexbox
    • Commonly Used Today
    • A one-dimensional flexible box module that lets us distribute content evenly and flexibly to fill all the available space.
  • CSS Grid
    • First true two-dimensional layout tool in CSS that lets us treat the viewport like a canvas.
    • Radically changes how we design page layouts for the web.

Flexbox (Flexible Box Layout Module)

  • A layout module for CSS that gives you an efficient way to arrange, organize, and size website elements to create responsive designs.
  • Lets us arrange items within a parent container and then get specific about how we want those items to behave in relationship to one another.

Flexbox

*CSS-Tricks: A complete Guide to Flexbox
  • Flex container (parent)
    • < section class="container >
      < /section >
  • Flex items (children) (elements within the parent container)
    • < section class="container >
      < /div >
      < /div >
      < /div >
      < /section >
    • The flex items are placed along flex lines, which are either a vertical or horizontal axis.
    • These items can be manipulated in layout, size, spaceing, and even change order along the flex line.
      *** You can manipulate the order of flex children without touching the HTML structure.
    • This allows you to take advantage of the available view port space and lets elements
      arrange themselves automatically as the view port dictates.
    • Alignment
    • Direction
    • Order
    • Size
  • Flexbox is a one dimensional presentation model. This means that layouts are processed in one dimension at a time, either as a row or a column.
  • You can define how space is distributed between elements and how they should be aligned in relationship to the parent container.

Flexbox Benefits

  • Responsive (mobile-friendly)
  • Ease of positioning child elements
  • Change presentation order without altering HTML
  • Control width and height based on parent element

Flexbox generally works best on small scale layouts.
