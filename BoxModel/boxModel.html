CSS Box Model

Every single element on your webpage is considered to be in a box: images, text, forms, everything.

Properties of Each Box

Properties relate to the size of the element AND how it relates to the elements around it.

  • Width
  • Height
  • Border
  • Margin
  • Padding

Content/Elements Edge/ Width & Height

  • If you don't specify a width or a height, then that's determined automatically based on how much space the content takes up.
  • You could also just specify the width and the height would be calculated automatically.
  • Example: The element is set to 100 pixels. By default, in the CSS Box Model, the box-sizing property is set to conent box. That means that if you set an element to 100 pixels, then just the conent box equals that 100 pixels. Any borders or padding would be added on top of that.

Padding

  • The space added to an element outside the element's edge.
  • Padding is clear space
  • You can define padding individually for the top, right, bottom, or left sides of an element.
  • When you add padding, you increase the size of the box, accordingly.
  • A box that has a content width of 100px, and padding of 20px on all sides would give us a total of 140 pixels of total width.
  • To change that behavior, you could change the box-sizing property to a border box, which tells the browser to account for any border and padding in the overall width that you specify for an element.

Border

  • Wraps around the padding
  • Factors into the overall width of the element (like padding does)
  • Border is visible and can have color and thickness applied to it
  • Can be applied specifically from top to bottom or right to left
  • Can be styled as solid, dotted, dashed, etc.

Margin

  • Represents the space outside the border
  • Used to define the space between elements on a page
  • Can be specified for each side of the element: top, right, bottom, left
  • Does not factor into the overall width of the element

Box Model Considerations

  • If a property is not declared, you cannot assume the value is 0
  • Elements often have default margins you need to account for - if you don't override them, the default value will be applied
  • A 100% width property, when combined with padding and vorders, can create elements that are larger than their parents

Normal Flow/Flow Layout

  • The default way elements are displayed on a page according to the HTML DOM
