CSS: Cascade

CSS Cascade Level 4 Specification

  • The cascade takes an unordered list of declared values for a given property on a given element,
  • sorts them by their declaration's precedence,
  • and outputs a single cascaded value.

The Cascade Algorithm

Here are the things the Cascade Algorithm checks, listed in order from highest weight to least weight:

  1. Origin and Importance
  2. Specificity
  3. Order of Appearance
  4. Inheritance

Origin and Importance

  • Highest weighted attribute
  • Inculdes:
    • user agent styles
    • browser overrides set by an individual user
    • CSS that you, as the developer, use
  • The importance of a CSS declaration is determined by the Important Syntax: !important
  • Adding !important to a CSS rule automatically jumps it to the front of the cascade algorithm

Specificity

  • The more specific selector wins.
  • A class selector is more specific than an element selector
  • An id selector is more specific than a class selector

Source Order

  • Refers to where the style rule is coming from (i.e. user agent, external style sheet, embedded style, inline style)
  • Refers also to the order that rules are written within a single style sheet.
    • Example:
      h1 {color: blue;}
      h1 {color: orange;}
    • These two rules could apply to the h1 element. The h1 ends up being colored orange because because both rules have an identical element selector and, therefore, carry the same specificity. But, the last one in the source order wins.

Inheritance

  • Some CSS property values set on parent elements are inherited by their child elements.
  • For example, font color or font family can be set on parent elements and inherited by child elements.
  • Child elements DON'T ALWAYS inherit property values from the parent.
  • For example, if we set a width of 50% on a parent element, we wouldn't want all descendants to be 50% of that width.

Calculating Specificity

  • Each CSS rule has a weight score based on the elements that make up the selector
  • The score is comprised of how many IDs are used, classes, and elements. And you get points respectively for each of those.
  • Specificity Calculator
  • ID score: 1, 0, 0 (can be read as 100)
  • class score: 0, 1, 0 (can be read as 10)
  • html element score: 0, 0, 1 (can be read as 1)
  • Combination Example: h1#sitetitle .logo: 1, 1, 1 (can be read as 111)
  • The highest specificity score wins, which make it the most specific of the bunch/li>

The Important Rule

!important

  • In CSS, you can apply the !important property to add more weight, or importance, to a property value pair than normal
  • Example: margin-left: 0 !important
  • We know that the below CSS code would return an h1 that is orange because the rule with orange is listed last.
    • h1 {color: blue;}
      h1 {color: orange;}
  • However, if we apply the !important property to the declaration of the first h1, the CSS code would return an h1 that is blue:
    • h1 {color: blue !important;}
      h1 {color: orange;}

    • When to Use !important

    • Override styles coming from code you don't control (third-party code)
    • Generally, do everything you can to avoid using !important.
      It's considered a bad practice because it breaks the natural cascade algorithm, making debugging much more difficult.

    !important Rules of Thumb

    • Always try to solve the problem with greater specificity before resorting to !important. Make better use of the cascade by using more specific rules.
    • Only use !important to overwrite CSS from an outside source.
    • Completely avoid !important when writing code for other developers to use.
    • Avoid !important on site-wide CSS.
